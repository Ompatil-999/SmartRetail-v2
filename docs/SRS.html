1. Introduction

1.1 Purpose

This document describes the Software Requirements Specification for Smart Retail, a web-based multi-store retail management system. It is intended for developers, testers, and project evaluators.

1.2 Scope

Smart Retail is a SaaS platform that allows retail store owners to manage inventory, customers, billing, and promotions through a unified web interface. Each store operates in complete data isolation using a store_id based multi-tenant architecture.

1.3 Definitions & Abbreviations

2. Overall Description

2.1 Product Perspective

Smart Retail is a standalone web application with a React.js frontend and Spring Boot backend communicating via RESTful APIs. Data is stored in MySQL with multi-tenant isolation.

2.2 User Classes

User Description
Store Owner Registers store, manages products, billing, customers, and offers
Walk-in Customer Purchases items (interacts indirectly through billing)

2.3 Operating Environment

Client: Modern web browser  |  Server: Java 17+ runtime  |  Database: MySQL 8.0

3. Modules Description (10 Modules)

Module 1: User Registration & Authentication

Store owner registers with store name, email, and password. JWT-based login with encrypted passwords (BCrypt). Secure logout with token invalidation.

Module 2: Store Management

View and update store profile including name, GST number, address, phone, and default tax rate for billing.

Module 3: Dashboard & Analytics

Displays 5 KPI cards — Total Products, Total Customers, Total Bills, Total Revenue, and Low Stock Count. Includes quick action buttons.

Module 4: Category Management

Create, read, update, and delete product categories. Prevents duplicate category names within the same store.

Module 5: Product Management

Full CRUD for products with name, price, barcode, stock quantity, and low stock threshold. Supports product image upload (JPG, PNG, WebP — max 2MB). Category assignment and search by name or barcode.

Module 6: Customer Management

Add, edit, and delete customers with name, email, and phone. Prevents duplicate emails within the same store. Searchable by name, email, or phone.

Module 7: Billing & POS

POS-style cart system with product search and add. Supports item-level and bill-level discounts. Automatic GST calculation. Inline customer creation during billing. Strict calculation order: Subtotal → Item Discount → Bill Discount → GST → Total. Automatic stock deduction after billing.

Module 8: Invoice Generation & Printing

Auto-generates unique invoice number (INV-{storeId}-{timestamp}). POS-style printable layout with store branding. QR code generated using ZXing library for verification. Includes customer details, item table, and billing breakdown. Print button opens native print dialog.

Module 9: Offer Management

Create promotional offers with title, description, discount percentage, and validity dates. Toggle active/inactive status. Automatic expired offer detection.

Module 10: Email Notifications

Asynchronous invoice email sent after billing (if customer has email). Professional HTML email template with store branding. Offer broadcast email to all customers. Non-blocking — does not delay billing flow.

12. Database Table Structure

Table: stores

Column Type Constraint
id BIGINT PRIMARY KEY, AUTO_INCREMENT
store_name VARCHAR(200) NOT NULL
gst_number VARCHAR(50)
default_tax_rate DECIMAL(5,2) DEFAULT 18.00
address TEXT
phone VARCHAR(20)
created_at DATETIME NOT NULL
updated_at DATETIME NOT NULL

Table: owners

Column Type Constraint
id BIGINT PRIMARY KEY, AUTO_INCREMENT
store_id BIGINT FOREIGN KEY → stores(id)
name VARCHAR(200) NOT NULL
email VARCHAR(200) UNIQUE, NOT NULL
password VARCHAR(255) NOT NULL
role VARCHAR(20) DEFAULT 'OWNER'
created_at DATETIME NOT NULL

Table: categories

Column Type Constraint
id BIGINT PRIMARY KEY, AUTO_INCREMENT
store_id BIGINT FOREIGN KEY → stores(id)
name VARCHAR(100) NOT NULL
description VARCHAR(500)
created_at DATETIME NOT NULL

Table: products

Column Type Constraint
id BIGINT PRIMARY KEY, AUTO_INCREMENT
store_id BIGINT FOREIGN KEY → stores(id)
category_id BIGINT FOREIGN KEY → categories(id)
name VARCHAR(200) NOT NULL
description VARCHAR(500)
price DECIMAL(12,2) NOT NULL
barcode VARCHAR(50)
stock_qty INT NOT NULL, DEFAULT 0
low_stock_threshold INT DEFAULT 10
image_url VARCHAR(500)
created_at DATETIME NOT NULL
updated_at DATETIME NOT NULL

Table: customers

Column Type Constraint
id BIGINT PRIMARY KEY, AUTO_INCREMENT
store_id BIGINT FOREIGN KEY → stores(id)
name VARCHAR(200) NOT NULL
email VARCHAR(200)
phone VARCHAR(20)
created_at DATETIME NOT NULL

Table: bills

Column Type Constraint
id BIGINT PRIMARY KEY, AUTO_INCREMENT
store_id BIGINT FOREIGN KEY → stores(id)
customer_id BIGINT FOREIGN KEY → customers(id)
bill_number VARCHAR(50) UNIQUE, NOT NULL
subtotal DECIMAL(12,2)
item_discount DECIMAL(12,2)
bill_discount DECIMAL(12,2)
taxable_amount DECIMAL(12,2)
tax_rate DECIMAL(5,2)
tax_amount DECIMAL(12,2)
total_amount DECIMAL(12,2)
status VARCHAR(20) DEFAULT 'COMPLETED'
created_at DATETIME NOT NULL

Table: bill_items

Column Type Constraint
id BIGINT PRIMARY KEY, AUTO_INCREMENT
bill_id BIGINT FOREIGN KEY → bills(id)
product_id BIGINT FOREIGN KEY → products(id)
product_name VARCHAR(200) NOT NULL
quantity INT NOT NULL
unit_price DECIMAL(12,2) NOT NULL
discount DECIMAL(12,2) DEFAULT 0
line_total DECIMAL(12,2) NOT NULL

Table: offers

Column Type Constraint
id BIGINT PRIMARY KEY, AUTO_INCREMENT
store_id BIGINT FOREIGN KEY → stores(id)
title VARCHAR(200) NOT NULL
description TEXT
discount DECIMAL(5,2) NOT NULL
valid_from DATE
valid_till DATE NOT NULL
active BOOLEAN DEFAULT TRUE
created_at DATETIME NOT NULL
